ubuntu ssh.cloudstudio.net: Permission denied (publickey).

之前在 windows 上直接就可以连接，但在ubuntu上却失败了.
和官方联系的过程学到一招， -vvvv​ 参数可以 debug ssh 连接过程
yaml
gs@box:~$ ssh -vvvv xxxx-xxx@xxx.ssh.cloudstudio.net OpenSSH_9.3p1 Ubuntu-1ubuntu3.2, OpenSSL 3.0.10 1 Aug 2023 ...省略大量日志... debug2: we did not send a packet, disable method debug1: No more authentication methods to try. xxx-xxx@xxx.ssh.cloudstudio.net: Permission denied (publickey).
最后是 ssh-keygen -t ed25519​ 重新生成了一个就好了，是我这ssh版本太老了。
后来我升级了一下 ssh 然后重新 ssh-keygen 也可以连接成功

[vite] Internal server error: Cannot read properties of undefined (reading 'send')

log
[vite] Internal server error: Cannot read properties of undefined (reading 'send') Plugin: @tailwindcss/vite:scan
我是在使用 tailwindcss 4.0.0-alpha.8​ 时遇到的这个问题，因为我的 vite 版本太低了是四点几， 没有实现 hot.send api ，所以hmr时就会报上面这个错误。手动升级到 ^5.1.6​ 就好了。

Subresource_Integrity 子资源完整性校验

https://developer.mozilla.org/en-US/docs/Web/Security/Subresource_Integrity
使用 内容交付网络 (CDN) 托管在多个站点之间共享的脚本和样式表等文件可以提高站点性能并节省带宽。 然而，使用 CDN 也存在风险，因为如果攻击者获得了 CDN 的控制权，则攻击者可以将任意恶意内容注入到 CDN 上的文件中（或完全替换文件），从而也可能攻击所有获取该内容的网站。来自该 CDN 的文件。
子资源完整性使您能够减轻此类攻击的一些风险，方法是确保您的 Web 应用程序或 Web 文档（从 CDN 或任何地方）获取的文件在第三方未向这些文件中注入任何其他内容的情况下进行交付 —并且没有对这些文件进行任何其他任何更改。
